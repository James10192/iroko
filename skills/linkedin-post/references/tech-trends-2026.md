- **Laravel 12** sorti fevrier 2025, support securite jusqu'en 2027

## Fintech / Mobile Money Afrique

- **Wave** : seul operateur mobile money non-telecom, stack in-house, mobile-first, 8 pays
- **BCG 2026** : "Beyond Payments — Africa's Second Fintech Wave" = credit, epargne, assurance via mobile
- **APIs** : Klasha MOMO Payout API (MTN, Airtel, Orange, Wave), GSMA Mobile Money API standard
- **Orange Money** : API Web Payment disponible en CI, Senegal, Mali, Cameroun, etc.
- **Programmable money** : telcos + fintechs + open-finance = argent programmable cross-border

**Angle post** : "En CI, la question n'est pas 'avez-vous un compte bancaire?' mais 'Wave ou Orange Money?'. Ca change completement comment on modelise les finances dans une app."

## Next.js / React

- **Next.js 16** stable avec App Router, Cache Components ('use cache'), proxy.ts
- **Turbopack** par defaut (plus Webpack)
- **Server Components** comme norme, Client Components uniquement pour interactivite
- **AI SDK 6** : tool calling, agents, streaming, MCP integration
- Tendance : integration IA native dans les apps React/Next.js

## PWA

- Toujours pertinent pour marches avec faible bande passante / appareils entry-level
- Service workers + VAPID push = alternative viable aux apps natives en Afrique
- Turbopack incompatible avec Serwist → service worker statique reste la solution
